# Weekly Summary + Improvements Design Spec

> Date: 2026-04-17
> Status: Draft
> Scope: Weekly Summary feature, lone surrogate sanitize, README rewrite

## 1. Weekly Summary Feature

### 1.1 Purpose

History 데이터를 기반으로 AI가 이번 주 활동을 5-6줄 서술형으로 요약.
Dashboard의 "honest historian" 정체성을 유지하면서, 주간 회고를 자연어로 제공.

### 1.2 Data Flow

```
getJourneyHistory()
  → filter: 이번 주 항목만 (weekStartDay 기준)
  → serialize: "project: title (description)" 형태
  → AI prompt (i18n 템플릿)
  → callClaude() → 서술형 텍스트
  → writeCache("weekly-summary", ...)
```

### 1.3 Weekly Range Calculation

```js
const weekDay = config.journey.weekStartDay ?? 1; // 0=Sun, 1=Mon
const today = new Date();
const diff = (today.getDay() - weekDay + 7) % 7;
const weekStart = new Date(today);
weekStart.setDate(today.getDate() - diff);
weekStart.setHours(0, 0, 0, 0);
// History items where date >= weekStart
```

### 1.4 AI Prompt (i18n)

**ko:**
```
이번 주({startDate}~{endDate}) 작업을 5~6줄로 요약해줘.
프로젝트명, 핵심 성과, 전체 흐름을 포함해줘.
톤: 따뜻한 회고, 함께 일한 동료가 돌아보는 느낌으로.
메시지만 출력.
```

**en (fallback):**
```
Summarize this week's work ({startDate}–{endDate}) in 5-6 lines.
Include project names, key achievements, and overall flow.
Tone: warm retrospective, like a colleague reflecting on shared work.
Output message only.
```

- Model: `config.claude.modelPriority` 따름 (기존 callClaude 패턴)
- i18n locale이 en/ko 외이면 en fallback

### 1.5 Cache & Scheduling

- **별도 캐시:** `weekly-summary` 캐시 파일 (journey 캐시와 독립)
- **갱신 주기:** `weekStartDay` 기준 주 1회 (기존 weekly cron에 추가)
- **API 합침:** `/api/journey` 응답에서 journey + weekly-summary를 합쳐 반환

```js
// server.js
app.get("/api/journey", async (_req, res) => {
  const journey = await getJourneyWithCache();
  const weeklySummary = await getWeeklySummaryWithCache();
  res.json({ ...journey, weeklySummary });
});
```

- `buildJourney()` 는 변경 없음 (history만 담당)
- `getWeeklySummaryWithCache()` — 캐시 유효하면 반환, 만료 시 AI 호출 후 저장
- weekly cron: `invalidateCaches("weekly-summary")` 추가

## 2. Lone Surrogate Sanitize

### 2.1 Problem

Claude CLI stdout에 이모지가 UTF-16 surrogate pair로 인코딩된 상태에서 truncation 되면 lone surrogate 잔존. 캐시/로그/claude-mem에 전파 시 API 400 에러 (`"no low surrogate in string"`).

### 2.2 Solution

`callClaudeWithModel()` (claude-gen.js:57) stdout 수신 직후 sanitize:

```js
function sanitizeSurrogates(text) {
  return text
    .replace(/[\uD800-\uDBFF](?![\uDC00-\uDFFF])/g, '')
    .replace(/(?<![\uD800-\uDBFF])[\uDC00-\uDFFF]/g, '');
}
```

적용 위치: `callClaudeWithModel()` 의 `stdout.trim()` 직후.

## 3. README Rewrite

### 3.1 Scope

README.md 전면 재작성:
- 프로젝트 소개 및 스크린샷
- 설치/실행 가이드
- 설정 가이드 (config.json)
- **예상 Token 사용량** — 각 AI 호출별 (profile, relationship, philosophy, voice, weekly-summary, avatar-decor, accent-color) input/output 예상치
- 아키텍처 다이어그램
- 라이선스

### 3.2 Screenshot

- Dummy data 기반 스크린샷
- 시간 표시: 00:00:00
- 기존 git 내 스크린샷 파일 삭제

### 3.3 Token Estimation

각 AI 호출의 프롬프트 + History 데이터를 분석하여 예상치 산출.
실제 측정은 구현 후 수행.
